# Track B 训练实验 · 实验设置与使用说明

> 目的：验证"网络学 deviation 方向"能否在 **fixed-budget** 下**打赢线搜索 FBHF / 惯性 FBHF / 动量 deviation**。这是 Track B（学习加速）唯一还没定的绿/红。
> 前置结论：探针3 已证 ℓ² 下降配额 deviation 有巨大天花板（千里眼 77×），但动量方向全失败 —— 所以必须真训练一个网络看能不能兑现。

## 一、一句话原理

$$
z_{k+1}=S_\gamma(z_k)+d_k,\qquad \|d_k\|^2\le \sigma\,\|z_k-x_k\|^2 .
$$

网络输出原始向量 → **硬投影**进上式的球。这个投影是 Track B 的安全入口：训练好坏不影响预算约束。正式手稿仍需把该预算条件对齐 FBHF-deviation 文献中的可吸收不等式，再声明弱收敛。

## 二、跑法（Google Colab，免费 T4 足够）

```bash
# 0) 上传 trackB_train.py，运行环境自带 torch/numpy/matplotlib
python trackB_train.py --selftest      # 先自检：算子伴随~1e-4以下 + 裸FBHF obj下降
python trackB_train.py --train         # 训练（默认 3000 步，T4 约 20–40 分钟）
python trackB_train.py --eval          # 载入 best.pt，对标基线，出表 + trackB_eval.png
```
无 GPU 也能跑（CPU，慢数倍）。先跑 `--selftest` 确认端口正确再训练。

## 三、关键配置（命令行可调）

| 参数 | 默认 | 含义 |
|---|---|---|
| `--size` | 96 | 图像 H=W（先 64 快验，再 96/128 出图） |
| `--K` | 40 | 训练展开步数（fixed-budget 预算；越大越贴"前中期加速"） |
| `--K_eval` | 400 | 评测步数 |
| `--batch` | 16 | 批大小 |
| `--iters` | 3000 | 训练步数 |
| `--eval_every` | 100 | 训练中验证与保存间隔 |
| `--sigma` | 0.5 | **deviation 预算** ‖d‖²≤σ‖z−x‖²（0.25 稳、0.9 激进） |
| `--lr` | 1e-3 | Adam 学习率 |

数据：在线生成分片常值图 + 随机高斯模糊(σ∈[1,2.5]) + 噪声(std∈[0.005,0.02])，训练/验证/测试用不同随机种子隔开（测试集 seed=2024 固定，不参与调参）。正则 μ=0.02 对所有方法一致，不单独调优。

## 四、网络与特征

- **输入 7 通道**：x、原始残差 x−x_r、原始动量 x−x_prev、对偶 y(2ch)、对偶残差 y−y_r(2ch)。全是迭代中可计算量，**不含真解/干净图**。
- **输出 3 通道**：d_x(1) + d_y(2)，经硬投影。
- 结构：3 层 3×3 卷积（~1.5 万参数）。轻量，防喧宾夺主。
- 损失：展开 K 步，末端加权 TV 目标 $F(x_k)=\tfrac12\|Kx_k-b\|^2+\mu\|Dx_k\|_{2,1}$ 之和（越晚权重越大）。**只用目标值，不用真解**，符合 §03 的 fixed-budget 口径。

## 五、评测基线与判据

评测在 held-out 测试集，统一初值 x₀=b、y₀=0，同预算跑 5 条：
1. 裸 FBHF（固定 γ=χ−η）
2. **线搜索 FBHF（免-Lipschitz）** ← 必须打赢
3. **惯性 FBHF（A20，小网格调最优 α,γ）** ← 必须打赢
4. 动量 deviation（可学基线，探针3 已知它失败）
5. **learned-dev（本文）**

指标：中位数相对原始误差 `‖x_k−x*‖/‖x*‖`（x* 长跑参考解）与 PSNR，随迭代画曲线，报 err@K、PSNR@K。

**绿灯判据**：learned-dev 在 err@K / PSNR@K 上**稳定优于线搜索与惯性**（不是只赢裸 FBHF、不是只看迭代数——按 §03 用 fixed-budget 质量）。
**红灯**：打不赢线搜索/惯性 → Track B 学习加速不成立，第一篇回落纯 A 轨（强收敛锚定 FBHF），学习最多作讨论。

## 六、正确性说明（沙箱无 GPU，未在此跑 torch）

- 本脚本算子是已验证 numpy 台架（探针 residual 1e-9）的直接 torch 端口。
- 唯一新构造（全网格高斯 OTF、D 的精确伴随）已在 numpy 中验证：`‖K‖=1`、`|<Kx,z>−<x,Ktz>|~1e-15`、`|<Dx,p>−<x,Dtp>|~1e-14`、`<Bz,z>~1e-14`（B 精确斜对称=精确单调）。
- 剩余风险仅 torch API 笔误，`--selftest` 会在 Colab 首跑时抓出（打印伴随误差与 obj 下降）。

## 七、建议节奏

1. `--selftest`（1 分钟）确认端口。
2. 先 `--size 64 --iters 800` 快训一版看 val gain 是否为正（趋势判断，10 分钟内）。
3. 正为正 → `--size 96 --iters 3000` 出正式结果 + `--eval` 出图。
4. 扫 `--sigma {0.25,0.5,0.9}` 与 `--K {30,40,60}`，报最优并列出失败设置（诚实）。
5. 打赢线搜索/惯性 → Track B 绿灯，可做第二篇主贡献或并入第一篇；否则第一篇纯 A 轨发。

> 提醒：这是"几天工时 + 几乎零算力成本"的验证。理论第一篇（A 轨强收敛定理）与本实验**并行**推进，互不阻塞。
